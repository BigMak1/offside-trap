МЕХАНИКА «OFFSIDE TRAP» — спецификация для этапа 2

КООРДИНАТЫ
- Поле: сетка 10 столбцов × 8 рядов. row 0 = верх = ВОРОТА. row 7 = низ = старт.
- Мяч стартует внизу по центру: клетка (row 7, col 4) сразу вскрыта и безопасна.

ЗАЩИТНИКИ (аналог «мин»)
- В части скрытых клеток сидят защитники с рейтингом подката: 1 (крайний), 2 (центральный), 3 (вратарь-свипер).
- Стартовый состав (в CONFIG, это балансится): всего 14 защитников — 7× рейтинг1, 5× рейтинг2, 2× рейтинг3.

ЧИСЛА «PRESSURE» (подсказка соседства)
- При вскрытии безопасной (без защитника) клетки на ней показывается СУММА рейтингов защитников в 8 соседних клетках (НЕ количество).
- pressure == 0 → каскадное авто-вскрытие соседей (flood-fill, как «0» в Сапёре).

ВСКРЫТИЕ = ДРИБЛИНГ (важная модель)
- Вскрывать можно ТОЛЬКО клетку, соседнюю (8 направлений) с уже вскрытой безопасной клеткой — игрок ведёт связную «дорожку» мяча снизу вверх. Именно это делает доход до ворот настоящим путём, а не лотереей по верхнему ряду.
- Первый ход всегда безопасен; стартовая клетка + её каскад дают начальную инфу.

ПОЕДИНОК (когда вскрыл клетку с защитником)
- У игрока: skill (старт 1) и stamina (старт 5).
- рейтинг ≤ skill → обыгрываешь БЕСПЛАТНО (0 стамины), клетка пройдена, +XP.
- рейтинг > skill → тратишь stamina = (рейтинг − skill), клетка пройдена, +XP.
- стамины не хватает оплатить → ПОТЕРЯ ВЛАДЕНИЯ = game over.
- XP за обыгранного защитника = его рейтинг.

ПРОКАЧКА
- Накопил XP до порога → skill +1. На level-up: stamina восстанавливается до максимума и max stamina +1 (по мотивам Dragonsweeper).
- Пороги XP в CONFIG, например нарастающие [3, 7, 12, 18, 25].

МАРКЕР (флаг)
- Правый клик / long-press / кнопка «mark mode» ставит конус на подозрительную клетку. Не вскрывает. Помеченную клетку нельзя вскрыть случайно (или требуй подтверждение).

ПОБЕДА / ПОРАЖЕНИЕ
- ПОБЕДА: вскрыл безопасную клетку в ряду ворот (row 0) → ГОЛ.
- ПОРАЖЕНИЕ: не можешь оплатить поединок (stamina < цены).

ОФСАЙДНАЯ ЛИНИЯ (скрытый тэлл — это НЕ штраф и НЕ условие проигрыша!)
- Офсайд здесь — это скрытый ПАТТЕРН РАЗМЕЩЕНИЯ защитников для дедукции, а не правило.
- При генерации задай скрытый offsideRow (например из {2,3}). Защитников «последней линии» (часть состава, особенно рейтинг 2–3) размещай рядом с offsideRow ± 1 с повышенной вероятностью.
- Смысл: когда игрок вскрыл пару высоких защитников и догадался о линии — клетки ВЫШЕ линии (ближе к воротам) с большей вероятностью безопасны. Это главное «озарение» игры.
- Для MVP можно сначала разместить защитников случайно, а офсайд-байас добавить вторым шагом как хук.

ГЕНЕРАЦИЯ И РЕШАЕМОСТЬ (критично)
- Доска генерируется детерминированно из даты по UTC (seeded RNG) — у всех одна ежедневная доска для шеринга. Добавь dev-override ?day=N.
- ГАРАНТИЯ: должна существовать связная безопасная дорожка от старта (row 7) до какой-то клетки ряда ворот (row 0). Сгенерировал → проверь, что путь есть → если нет, перегенерируй с другим сидом.
- Идеал (если осилишь): прогоняй простой солвер и гарантируй, что путь находится ЛОГИКОЙ без угадывания. Если для MVP тяжело — минимум гарантируй существование безопасного пути + каскад на первом ходу.

ТЕХ-ТРЕБОВАНИЯ
- Все числовые параметры (размер поля, состав защитников, старт stamina/skill, пороги XP, диапазон offsideRow) — в одном объекте CONFIG, чтобы я мог балансить.
- Отдели ЛОГИКУ от РЕНДЕРА: вскрытие/поединок/каскад считай как изменение состояния (верни список «что произошло»), потом анимируй рендером — это ложится на пиксельный пайплайн этапа 2 и проще дебажить.
- Добавь debug-режим: показать всю доску (защитников и числа) для теста.
